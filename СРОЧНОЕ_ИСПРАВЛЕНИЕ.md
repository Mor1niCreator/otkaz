# 🔧 СРОЧНОЕ ИСПРАВЛЕНИЕ

## Проблема: "next" не является командой

Это означает, что зависимости не установлены или повреждены.

## ✅ РЕШЕНИЕ:

### Шаг 1: Удалите node_modules

```powershell
Remove-Item -Recurse -Force node_modules
Remove-Item package-lock.json
```

### Шаг 2: Очистите кеш npm

```powershell
npm cache clean --force
```

### Шаг 3: Установите зависимости заново

```powershell
npm install
```

Это займёт 1-3 минуты.

### Шаг 4: Инициализируйте базу данных

```powershell
npm run db:init
```

### Шаг 5: Запустите сервер

```powershell
npm run dev
```

## 🎯 Если не помогло:

### Вариант А: Проверьте Node.js

```powershell
node --version
npm --version
```

Должно показать версии. Если нет - переустановите Node.js.

### Вариант Б: Установите вручную

```powershell
npm install next@14.2.0 react@18.3.0 react-dom@18.3.0
npm install
npm run dev
```
