## 🔍 Что было не так:

При регистрации API НЕ возвращал поля `currency` и `language`, поэтому в localStorage они были `undefined`.

Когда вы пытались создать запись, передавался `currency: undefined` → ошибка 400.

---

## ✅ Что исправлено:

1. **API регистрации** теперь возвращает все поля:
   - currency (USD по умолчанию)
   - language (en по умолчанию)
   - points
   - rank

2. **Calendar** теперь использует USD по умолчанию если currency отсутствует

3. **API создания записей** теперь показывает какие поля отсутствуют

## 💡 Альтернатива (если не хотите перерегистрироваться):

Откройте Console в браузере (F12) и выполните:

```javascript
const user = JSON.parse(localStorage.getItem('user'));
user.currency = 'USD';
user.language = 'en';
localStorage.setItem('user', JSON.stringify(user));
location.reload();
```
